# 260706-시장별 DB 분리 및 용량 최적화 계획

본 계획서는 단일 로컬 데이터베이스 파일(`db/60day_high_yf.csv`)에 수천 개 종목의 주가 내역이 누적되어 파일 용량 증가, 입출력 속도 저하, Git 푸시 지연 등의 부하가 발생하는 문제를 진단하고, 이를 최적화하기 위한 설계 및 이행 계획을 제시합니다.

---

## 1. 개요 및 문제 진단

### A. 현재 상황
* 분석 대상 시장: 미국(S&P 500, NASDAQ), 중국(SSE, SZSE), 대만(TWSE - 지수)
* 전체 분석 대상 종목 수: 약 8,500+개 (미국 3,500+, 중국 5,000+, 대만 1)
* 수집 데이터 기간: 각 종목당 최신 150일 분량
* 데이터 행수 계산: `8,500개 종목 * 150일 = 1,275,000+ 행`
* 단일 CSV 파일(`db/60day_high_yf.csv`)의 크기가 **35MB 이상**으로 커지며, 매 실행 시마다 이를 로드 및 가공하고 Git에 다시 푸시하는 과정에서 큰 오버헤드가 발생합니다.

---

## 2. 제안하는 최적화 방향

> [!TIP]
> **시장별 분리(Option A)**와 **슬라이딩 윈도우 제한(Option C)**을 동시에 적용하는 **하이브리드 아키텍처**를 권장합니다.

* **방안 1: 시장별 데이터베이스(CSV) 분리 (추천)**
  * 하나의 거대한 `60day_high_yf.csv` 파일을 각 시장군에 따라 분리하여 보관합니다.
    - `db/60day_high_nasdaq.csv` (약 15MB)
    - `db/60day_high_sp500.csv` (약 2MB)
    - `db/60day_high_sse.csv` (약 8MB)
    - `db/60day_high_szse.csv` (약 10MB)
    - `db/60day_high_twse.csv` (약 2KB - 단일 지수)
  * **기대 효과**:
    - 개별 입출력 크기가 30% 이하로 감소하여 판다스 파일 읽기/쓰기 속도가 대폭 향상됩니다.
    - Git 버전 관리 시 변경된 시장의 파일만 동기화되므로 네트워크 전송 부하가 획기적으로 낮아집니다.

* **방안 2: 슬라이딩 윈도우 적용 (Pruning) (추천)**
  * 60일 신고가 산출을 위해 필요한 최소 과거 데이터는 영업일 기준 약 60~80일(달력 기준 약 90~120일) 수준입니다.
  * DB 업데이트 완료 시점에 **최근 100일(또는 120일) 이전의 과거 데이터는 정기적으로 삭제(Pruning)**하여 DB 파일의 크기가 무한히 증가하는 것을 방지합니다.
  * **기대 효과**:
    - 데이터 누적이 진행되어도 파일 크기가 항상 임계점 이하(예: 약 100일분)로 유지되어 평생 일정한 응답 속도를 보장합니다.

## 3. 상세 구현 계획

### A. 대상 파일
* `src/get_60day_high_yf.py`

### B. 변경할 주요 로직
1. **DB 경로 동적 정의**:
   ```python
   def get_db_file_path(market_name):
       return os.path.join(BASE_DIR, "db", f"60day_high_{market_name.lower()}.csv")
   ```
2. **시장별 데이터 로드 및 맵핑 루프화**:
   * `update_and_get_data`에서 각 시장별로 로컬 DB를 읽고, `yfinance`를 호출한 뒤, 각 파일별로 저장합니다.
3. **슬라이딩 윈도우 Pruning 로직 적용**:
   * 각 시장별 저장 직전에 아래와 같이 필터링을 수행합니다:
     ```python
     # 최근 100일 영업일 기준에 속하는 날짜들만 슬라이싱하여 보존
     unique_dates = sorted(df_market["날짜"].unique())
     if len(unique_dates) > 100:
         cutoff_date = unique_dates[-100]
         df_market = df_market[df_market["날짜"] >= cutoff_date]
     ```
